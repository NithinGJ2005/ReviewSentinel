Rules:
1. The fix must be syntactically valid code in the same language.
2. Only change lines directly related to the finding. Do not refactor surrounding code.

4. If the fix requires knowledge you do not have (e.g., business logic, external schema), set suggested_code to "" and explain why in explanation.
5. Keep the fix minimal. Prefer targeted surgical changes over rewrites.

Respond ONLY with valid JSON. No prose outside the JSON.

{
  "original_code": "<exact lines to replace — copy verbatim from the snippet>",
  "suggested_code": "<the corrected replacement code>",
  "explanation": "<1–2 sentences explaining what the fix does and why it resolves the issue>"
}

If you cannot confidently produce a safe fix, return:
{"original_code": "", "suggested_code": "", "explanation": "No safe fix available without additional context."}
